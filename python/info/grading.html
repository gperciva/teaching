Lab assessment

Grades

Lab material will be graded by the main lab demonstrator. There are 5 marks for each graded lab. The criteria is split into three categories:

  • 2 marks: code meets specifications.

    (0 = none, 1 = partial, 2 = all)

  • 2 marks: student understands and can explain the code.

    (0 = cannot explain, 1 = can answer simple questions, 2 = good understanding)

  • 1 mark: good code style.

    (0 = poor style, 1 = good style)

Rationale

More than 50% (some people estimate 80-90%!) of a programmer's job is reading code, not writing it. Maintaining a large code base is a difficult task even if the code was written well; if it was written poorly, it becomes almost impossible. In addition, the easier your code is to read, the fewer mistakes you will make in the first place!
